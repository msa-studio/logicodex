# Logicodex Streaming Semantic Compiler — v1.31.0-alpha

> **"A compiler that stores only the semantic essence it needs."**

Architect: Mohamad Supardi Abdul (mymsastudio@gmail.com)

---

## Overview

v1.31.0-alpha memperkenalkan **Tier 2: Streaming Semantic Compiler** — transformasi seni bina yang menukar Logicodex dari compiler tradisional (simpan AST penuh dalam RAM) ke compiler generasi seterusnya (stream function-by-function, discard AST selepas codegen).

**Problem:** Compiler tradisional simpan AST penuh dalam RAM. Untuk program 1M baris, ini boleh memakan GB memori.

**Solution:** 2-Pass Engine — Pass 1 scan ringan (metadata), Pass 2 stream mendalam (analyze + codegen + discard).

**Result:** RAM kekal rata tak kira saiz program. 100 fungsi = < 20KB metadata.

---

## 3-Tier Architecture

```
Tier 1 (Parser)   → Full AST per function (TEMPORARY — discarded after Pass 2)
Tier 2 (This PR)   → MetadataGraph + SemanticSummary (PERSISTENT — lightweight)
Tier 3 (Codegen)   → LLVM IR chunks (STREAMED — one function at a time)
```

**Flow:**
```
Program → Pass 1 (pre-declare) → MetadataGraph → Pass 2 (stream) → Results
              ↑ lightweight index       ↑ deep analysis per function
                                         ↑ discard AST after each function
```

## SemanticSummary (~64 bytes per symbol)

```rust
pub struct SemanticSummary {
    symbol_id: u32,           // Unique ID
    name: String,             // Symbol name
    params: Vec<Type>,        // Parameter types
    ret_type: Option<Type>,   // Return type
    effects: Capability,      // Pure? IO? Unsafe? Concurrent?
    inline_cost: InlineCost,  // Trivial/Small/Medium/Large/Recursive
    is_actor: bool,           // Is this an actor?
    channels_used: Vec<String>, // For actors: channels used
    is_recursive: bool,       // Self or mutual recursion
    callees: Vec<u32>,        // Symbol IDs this calls
}
```

**Size:** ~64 bytes vs. **thousands** of bytes for full AST.

**100 functions in MetadataGraph = < 20KB.**

---

## Capability (u8 bitflags)

```
PURE       = 0x01   — No side effects, safe to inline
IO         = 0x02   — Performs I/O
UNSAFE     = 0x04   — Contains unsafe operations
CONCURRENT = 0x08   — Uses actors/channels
HARDWARE   = 0x10   — Hardware zone access
DIVERGING  = 0x20   — May not terminate
```

Default: `PURE` — assume safe until proven otherwise.

## InlineCost

| Cost | Criteria | Inline Decision |
|---|---|---|
| `Trivial` | 1 statement | Always inline |
| `Small` | 2-3 statements | Inline candidate |
| `Medium` | 4-10 statements | Inline if hot |
| `Large` | > 10 statements | Don't inline |
| `Recursive` | Self/mutual recursion | Never inline |

## 2-Pass Engine

### Pass 1: Pre-Declaration (Lightning Scan)

```rust
pub fn pass1_predeclare(program: &Program) -> Result<MetadataGraph, SemanticError>
```

- Scan seluruh program dalam satu laluan pantas
- Kumpul signatures fungsi, deklarasi actor, topologi channel
- Bina MetadataGraph (index ringan)
- Detect mutual recursion melalui call graph edges

### Pass 2: Deep Streaming

```rust
pub fn pass2_streaming(program: &Program, graph: &mut MetadataGraph, mode: CompileMode)
    -> Result<StreamingResult, SemanticError>
```

- Proses setiap fungsi satu-per-satu
- Infer capabilities dari body
- Estimate inline cost
- **DISCARD AST selepas setiap fungsi**

### CompileMode

| Mode | Behavior | Use Case |
|---|---|---|
| `Pantas` | Aggressive streaming, immediate discard | Development (fast iteration) |
| `Pakar { max_ram_mb }` | Full optimization within memory budget | Production (deterministic memory) |
