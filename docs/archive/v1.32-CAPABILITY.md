## Overview

v1.32.0-alpha memperkenalkan **Capability-Based Security Architecture** — sistem keupayaan statik yang memastikan keamanan sistem **pada masa kompil** (compile-time), bukan runtime. Ini mengelakkan overhead semakan runtime dan menjamin zero runtime mediation.

**Prinsip Teras:**
- **Gate** = Kontrak compile-time (Type-Level). Menentukan SIAPA boleh akses APA.
- **Door** = Pengangkutan runtime (SPSC / DirectCall). Menentukan BAGAIMANA data bergerak.
- **Zero Runtime Mediation** = Semua semakan pada masa kompil. Kalau compile lulus, program proven safe.

---

## Architecture: Gate vs Door

```
┌─────────────┐      ┌──────────────┐      ┌─────────────┐
│   Module A  │──────│     Gate     │──────│   Module B  │
│  (Provider) │      │  (Contract)  │      │  (Consumer) │
└─────────────┘      └──────────────┘      └─────────────┘
       │                                           │
       │          ┌──────────────┐                │
       └─────────→│    Door      │←───────────────┘
                  │  (Transport) │
                  └──────────────┘
```

| Layer | Concept | Runtime Cost |
|---|---|---|
| **Gate** | Compile-time capability contract | **ZERO** — validated at compile time |
| **Door** | Runtime transport (SPSC / DirectCall) | Normal — no extra checks |

## Three Gate Types

Mengelakkan "mega gate" / ioctl trap yang menjadi masalah dalam sistem OS tradisional.

| Type | When | Example | Inline-able |
|---|---|---|---|
| `DirectCall` | Inline-able sync | Math, Crypto | ✅ Yes |
| `Message` | Async SPSC | Sensor, Network I/O | ❌ No |
| `Hardware` | Bare-metal only | GPIO, DMA, Timer | ❌ No (exclusive) |

---

## Standard Gate Domains

### Storage — Operasi fail
- `Storage.Baca` — Read files
- `Storage.Tulis` — Write/create files
- `Storage.Padam` — Delete files

### Net — Operasi rangkaian
- `Net.Send` — Send data (Message gate)
- `Net.Recv` — Receive data (Message gate)
- `Net.Raw` — Raw socket (Hardware gate — **sensitive**)

### UI — Antaramuka pengguna
- `UI.Papar` — Display output
- `UI.Input` — Read input

### HW — Hardware bare-metal (eksklusif)
- `HW.GPIO` — General Purpose I/O
- `HW.Timer` — Hardware timers
- `HW.DMA` — Direct Memory Access
- `HW.ADC` — Analog-to-Digital Converter

### Audio — Audio processing
- `Audio.Main` — Play audio
- `Audio.Rakam` — Record audio

### Crypto — Kriptografi (selalu DirectCall)
- `Crypto.Hash` — Cryptographic hash
- `Crypto.Encrypt` — Encryption

## Usage Example

```ldx
// Module A provides Storage gates
function driver_storage() {
    // PROVIDE: Storage.Baca, Storage.Tulis
}

// Module B requires Storage gate
function app_file_manager() {
    // REQUIRE: Storage.Baca
    let data = open("config.txt")
}
```

**Topology verification:** `app_file_manager → Storage.Baca ← driver_storage` ✅ VALID

## GateContract

Setiap modul mengisytiharkan kontrak:
```rust
pub struct GateContract {
    module_name: String,
    provides: Vec<GateRef>,   // Gates yang disediakan
    requires: Vec<GateRef>,   // Gates yang diperlukan
}
```

## CapabilityTopology

Graph yang memetakan:
- **WHO** (symbol) → **CAN DO WHAT** (gate)

### Verifikasi
```
Untuk setiap REQUIRE → mesti ada PROVIDE
Kalau takde → CapabilityContractViolation (compile error)
```

### Supply-Chain Security: `.cap` File

Setiap compile menghasilkan fail `.cap` (Capability Topology Graph):
```cap
# Logicodex Capability Topology Graph v1.0
## PROVIDERS
Storage.Baca <- driver_storage
Storage.Tulis <- driver_storage
## CONSUMERS
Storage.Baca -> app_file_manager
## CONTRACTS
MODULE driver_storage
  PROVIDE Storage.Baca
  PROVIDE Storage.Tulis
```

### Capability Diff

Banding 2 versi `.cap` untuk detect **privilege escalation** (macam XZ Utils attack):
```
OLD: app → Storage.Baca
NEW: app → Storage.Baca, Net.Raw  ← PRIVILEGE ESCALATION DETECTED!
```

**Gate sensitif yang trigger escalation:**
- `Net.Raw` — raw socket access
- `HW.*` — any hardware gate
- `Storage.Padam` — delete operations
