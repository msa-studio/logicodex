# Logicodex v1.34.0-alpha — Sharded Deterministic Reactor

> "Shard Isolation: Setiap CPU Core = satu ReactorInstance + LocalPool. Cross-Shard = Door Only."

## Overview

The Sharded Deterministic Reactor extends the v1.33 Network Reactor with compile-time static sharding. Each CPU core runs exactly one reactor instance with a dedicated local memory pool. Services are statically mapped to cores at compile time, and cross-shard communication is only permitted through SPSC Doors.

## Components

### ShardTopology (`src/tier2/shard.rs`)

The compile-time topology that defines all shards, services, and cross-shard doors.

```rust
pub struct ShardTopology {
    pub assignments: Vec<ShardAssignment>,
    pub service_graph: ServiceGraph,
    pub doors: Vec<DoorRef>,
}
```

#### ShardAssignment

```rust
pub struct ShardAssignment {
    pub shard_id: u32,      // 0, 1, 2, ...
    pub core_id: u32,       // CPU core pinned
    pub services: Vec<String>, // Service names
    pub budget_mb: u32,     // Memory budget
    pub gates: Vec<GateRef>, // Allowed gates
}
```

#### ServiceGraph + ServiceNode

```rust
pub struct ServiceNode {
    pub name: String,
    pub port: u16,
    pub requires: Vec<GateRef>,
    pub handler: String,
    pub policy: String,
    pub assigned_shard: Option<u32>,
}
```

#### CommEdge + CommType

- `CommType::Door` — SPSC, valid for cross-shard
- `CommType::Direct` — Intra-shard only, compile-time forbidden cross-shard

#### DoorRef

Cross-shard SPSC channel reference:

```rust
pub struct DoorRef {
    pub from_shard: u32,
    pub to_shard: u32,
    pub message_type: String,
    pub capacity: usize,
}
```

### Verification (5 Checks)

`ShardTopology::verify()` performs these compile-time checks:

1. **UnassignedService** — Every service must be assigned to exactly one shard
2. **DuplicateAssignment** — No service in more than one shard
3. **ForbiddenDirectCrossShard** — Cross-shard communication must use Door
4. **BudgetOverflow** — Estimated memory must not exceed shard budget
5. **EmptyShard** — Every shard must have at least one service
6. **CoreConflict** — No two shards pin to the same core

## ShardedReactor (`src/net/sharded_reactor.rs`)

Runtime component: `Vec<ShardInstance>` with per-core event loops.

## ShardLocalPool (`src/net/shard_local_pool.rs`)

Per-shard memory pool with budget tracking.

## CPU Affinity (`src/net/affinity.rs`)

Wrapper for `sched_setaffinity` — enforces static compile-time mapping.

## Example Manifest

```ldx
shard WebShard { core: 0, services: [WebServer, ApiGateway], budget_mb: 256 }
shard MetricsShard { core: 1, services: [MetricsCollector], budget_mb: 64 }
door WebShard -> MetricsShard { message_type: MetricPayload, capacity: 1024 }
```
