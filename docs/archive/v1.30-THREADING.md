## Overview

v1.30.1-alpha memperkenalkan **actor-model concurrency** ke dalam bahasa Logicodex melalui 3 fasa pembangunan. Sistem ini terinspirasi oleh model aktor (Erlang/Akka) tetapi dioptimumkan untuk sistem rendah-latensi dengan **zero-copy ownership transfer**.

Prinsip teras:
- **Kotak в†’ Actor** = satu unit konkurensi (1 OS thread)
- **Pintu в†’ Channel** = saluran komunikasi SPSC (single producer, single consumer)
- **Zero-Copy** = ownership dipindahkan, bukan data diduplikasi
- **Compile-Time Verification** = topology dan ownership diverifikasi sebelum runtime

## 3 Fasa Pembangunan

### Phase 1: Threading Foundation (PR #27)

Komponen teras actor-model.

```ldx
actor Producer {
    let out: Channel<Producer, Consumer, i32>
    out.send(42)
}

actor Consumer {
    let in_ch: Channel<Producer, Consumer, i32>
    let msg = in_ch.recv()
}

spawn Producer()
spawn Consumer()
join Producer
join Consumer
```

**Keywords:**
| Keyword | English | Purpose |
|---|---|---|
| `actor` | actor | Declare concurrency unit |
| `channel` | channel | SPSC communication channel |
| `spawn` | spawn | Create actor instance |
| `send` | send | Send value through channel |
| `recv` | receive | Receive value from channel |
| `join` | join | Wait for actor completion |

**AST Nodes:** `Stmt::Actor`, `Type::Channel`, `Expr::Spawn`, `Expr::Send`, `Expr::Recv`, `Expr::Join`

**Semantic Checks:**
- `ActorNotFound` вЂ” spawn non-existent actor
- `DuplicateActor` вЂ” duplicate actor definition
- `InvalidChannelTopology` вЂ” channel endpoint mismatch
- `SpawnNonActor` вЂ” spawn on non-actor name

**Files:** `src/ast.rs`, `src/lexer.rs`, `src/parser.rs`, `src/semantic.rs`, `lib/core/thread.ldx`, `lib/core/sync.ldx`

### Phase 2: Zero-Copy Ownership Transfer (PR #28)

RAII move semantics melalui channel.

```ldx
actor Producer {
    let data = 42
    let processed = data * 2
    out.send(processed)     // ownership MOVED
    // let x = processed    // ERROR: UseAfterSend
}
```

**Ownership Rules:**
1. `send(variable)` вЂ” variable marked as **moved**
2. Using moved variable в†’ `UseAfterSend` error at compile time
3. `send(literal)` or `send(expression)` вЂ” no ownership transfer
4. Double-send same variable в†’ compile error

**Semantic Checks:**
- `UseAfterSend { name }` вЂ” variable already sent through channel

**Files:** `src/semantic.rs` (`moved_via_channel`), `src/codegen.rs` (Release/Acquire stubs), `lib/core/ring_buffer.ldx`

### Phase 3: Backpressure + Scheduler (PR #30)

Non-blocking operations + cooperative scheduler.

```ldx
actor Producer {
    while (count < 10) {
        let ok = out.try_send(count)
        if (ok.is_ok()) {
            count = count + 1
        } else {
            yield()  // Give control to other actors
        }
    }
}
```

**New Expressions:**
| Expression | Syntax | Return Type |
|---|---|---|
| `try_send` | `channel.try_send(value)` | `Result<bool, IoError>` |
| `try_recv` | `channel.try_recv()` | `Option<T>` |
| `yield` | `yield()` | `Unit` |
| `sleep` | `sleep(ms)` | `Unit` |
| `timeout_recv` | `channel.timeout_recv(ms)` | `Result<T, RecvTimeout>` |

**Error Types:**
- `ChannelFull { name }` вЂ” channel buffer full
- `RecvTimeout { name, timeout_ms }` вЂ” receive timed out

**Files:** `lib/core/scheduler.ldx` (cooperative round-robin scheduler)

## Ring Buffer (SPSC)

```
Producer:  write data в†’ Release write to tail
Consumer:  Acquire read from head в†’ read data
```

Functions:
- `ring_new<T>(capacity)` вЂ” allocate power-of-2 buffer
- `ring_send<T>()` вЂ” Producer write (Release)
- `ring_recv<T>()` вЂ” Consumer read (Acquire)
- `ring_try_send<T>()` вЂ” non-blocking with backpressure
- `ring_try_recv<T>()` вЂ” non-blocking receive
- `ring_timeout_recv<T>()` вЂ” receive with timeout
