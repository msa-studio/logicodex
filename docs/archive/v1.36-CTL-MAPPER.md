# Logicodex v1.36.0-alpha — CTL Mapper (Fasa B)

> "Auto-generate WIT from CapabilityGraph — 6 domain mappings."
>
> "Project INTO, not borrow FROM."

## Overview

The CTL (Capability Translation Layer) Mapper is the bridge between Logicodex's capability-native world and the WASM ecosystem. It reads a CapabilityGraph IR (v1.35) and produces:

1. **WIT** (WebAssembly Interface Types) — for WASM component linking
2. **Populated `CapabilityRef.wit_mapping`** — for runtime host resolution
3. **Host reactor stubs** — for HW gates that need host-side mediation

## 6 Domain Mappings

| Logicodex Domain | WASI Target | Gate Type | HW? |
|---|---|---|---|
| `Storage` | `wasi:filesystem` | DirectCall | No |
| `Net` | `wasi:sockets` | Message | No |
| `UI` | `wasi:cli` | DirectCall | No |
| `HW` | `logicodex:host-reactor` | Hardware | **Host-mediated** |
| `Audio` | `wasi:io/custom` | Message | No |
| `Crypto` | `wasi:crypto` | DirectCall | No |

## Architecture

```
CapabilityGraph IR ──► CtlMapper ──► WIT Output
(v1.35)                  │
                         ├──► map_capability() ──► CapabilityRef.wit_mapping
                         ├──► map_graph() ──► populate all refs
                         ├──► generate_wit() ──► complete WIT document
                         └──► generate_host_reactor_stub() ──► Rust host code
```

## Core Types

### WitDomain

```rust
pub enum WitDomain {
    WasiFilesystem,  // Storage → wasi:filesystem
    WasiSockets,     // Net → wasi:sockets
    WasiCli,         // UI → wasi:cli
    HostReactor,     // HW → logicodex:host-reactor
    WasiIoCustom,    // Audio → wasi:io/custom
    WasiCrypto,      // Crypto → wasi:crypto
    Unknown(String), // Fallback
}
```

### WitOperation

WIT function signature with parameters and return type:

```rust
pub struct WitOperation {
    pub logicodex_op: String,  // "Baca"
    pub wit_op: String,        // "read"
    pub params: Vec<(String, String)>, // [("path", "string"), ("offset", "u64")]
    pub returns: Option<String>, // "result<list<u8>, error>"
}
```

Example generated signature:
```wit
  read: func(path: string, offset: u64) -> result<list<u8>, error>;
```

### CtlMapper

```rust
pub struct CtlMapper {
    mappings_applied: usize,
    hw_gates: Vec<CapabilityRef>,        // Detected HW gates
    unknown_domains: Vec<String>,        // Unknown domains found
    overrides: HashMap<String, String>,  // Manual overrides
}
```

## API

### Map a single capability

```rust
let mut mapper = CtlMapper::new();
let mut cap = CapabilityRef::new("Storage", "Baca", GateType::DirectCall);
mapper.map_capability(&mut cap);
// cap.wit_mapping == Some("wasi:filesystem/read")
```

### Map entire graph

```rust
let mut graph = CapabilityGraph::new(CompileTarget::Wasm);
// ... add services, shards ...
let mut mapper = CtlMapper::new();
mapper.map_graph(&mut graph); // Populates all wit_mapping fields
```

### Generate WIT

```rust
let wit = mapper.generate_wit(&graph);
// Complete WIT document with world + interfaces
```

### Manual overrides

```rust
mapper.add_override("Storage.Baca", "my-custom:custom-read");
// All Storage.Baca refs will use "my-custom:custom-read" instead of auto-mapping
```

### One-shot pipeline

```rust
let (wit, stats) = map_and_generate_wit(&mut graph);
println!("{}", stats.summary());
```

## HW Gate Handling

HW gates are NEVER exposed to the WASM guest. They are detected and routed through the Host Reactor:

```rust
// HW.GPIO → host-reactor stub
let mut cap = CapabilityRef::new("HW", "GPIO", GateType::Hardware);
mapper.map_capability(&mut cap);
// cap.wit_mapping == Some("logicodex:host-reactor/gpio-control ...")
```

Generated host reactor stub:
```rust
pub fn host_reactor_hw_gpio(reactor: &mut HostReactor, pin: u32, mode: String) -> Result<u32, ReactorError> {
    reactor.with_hardware_zone(|hw| {
        hw.gpio(pin, &mode)
    })
}
```

## Unknown Domain Handling

Domains not in the 6 standard mappings get `logicodex:custom`:

```rust
let mut cap = CapabilityRef::new("CustomDomain", "DoIt", GateType::DirectCall);
mapper.map_capability(&mut cap);
// cap.wit_mapping == Some("logicodex:custom/doit")
```
