> **"No Global Lock, No Hidden State, No Magic."**

## Overview

v1.33.0-alpha memperkenalkan **Deterministic Network Reactor** вҖ” sistem rangkaian event-driven dengan jaminan deterministik melalui RAII auto-cleanup dan taint state machine.

**Killer Feature:** Setiap koneksi = objek RAII. Bila drop вҶ’ `close(fd)` automatik. Tiada socket leak. Tiada GC. 100% deterministik.

**Architecture:** Door + Gate + Service
- **Door** (SPSC Ring Buffer) = Data Transport вң… v1.30
- **Gate** (Capability Contract) = Compile-time security вң… v1.32
- **Service** (Port-based Actor) = Event Loop + Handler вң… v1.33

---

## Service Declaration

Tiada lagi `bind()`, `listen()`, `accept()` manual. Semua melalui manifest deklaratif:

```ldx
service WebServer {
    port: 443,
    requires: Net.Admin,
    handler: WebHandler,
    policy: Block,
}

service GameServer {
    port: 7777,
    handler: GameHandler,
    policy: DropOldest,
}
```

**Compiler Magic:**
1. Daftarkan servis ke dalam Main Gate (Reactor)
2. Auto-shutdown port yang tak didaftarkan
3. Percubaan capaian ke port luar topologi вҶ’ ditolak di peringkat driver

## RAII Connection вҖ” Killer Feature

```rust
pub struct Connection {
    fd: i32,                    // Opaque вҖ” user cannot access directly
    port: u16,
    service_name: String,
    taint: TaintState,          // Healthy вҶ’ Suspicious вҶ’ Closing
    bytes_received: u64,
    bytes_sent: u64,
    consecutive_errors: u32,
    last_activity_ms: u64,
}

impl Drop for Connection {
    fn drop(&mut self) {
        self.shutdown();        // close(fd) deterministik
        // Tiada socket leak. Tiada GC. 100% deterministik.
    }
}
```

**Jaminan:**
- Koneksi tamat (timeout) вҶ’ objek di-drop вҶ’ `close(fd)`
- Koneksi terputus вҶ’ objek di-drop вҶ’ `close(fd)`
- Koneksi malicious вҶ’ objek di-drop вҶ’ `close(fd)` + padam rekod Reactor
- Tiada menunggu GC atau async runtime berat

## Backpressure Policies

Apabila buffer servis penuh:

| Policy | Behavior | Use Case |
|---|---|---|
| `Block` | Tunggu buffer kosong | Paling selamat, latency tinggi |
| `DropOldest` | Buang data paling lama | Real-time streaming (sensor) |
| `Error` | Kembalikan error (retry) | Protocol reliable (banking) |

### Presets

```rust
PolicyConfig::default()          // 64KB buffer, 30s timeout, 1024 max conn
PolicyConfig::high_throughput()  // 256KB buffer, 5s timeout, 4096 max conn  
PolicyConfig::critical()         // 16KB buffer, 10s timeout, 256 max conn
```

**Events:**
- `Readable` вҖ” Data tersedia
- `Writable` вҖ” Sedia menulis
- `Error` вҖ” Error berlaku
- `Hangup` вҖ” Peer tutup koneksi
- `Closed` вҖ” Koneksi ditutup

**Actions:**
- `Keep` вҖ” Terus pantau fd
- `Close` вҖ” Tutup fd + buang (RAII)
- `Reregister` вҖ” Tukar interest (read/write)
