# Logicodex v1.35.0-alpha — CapabilityGraph IR (Fasa A)

> "Single Source of Truth — language-agnostic capability representation."
>
> "Project capability model INTO the WASM ecosystem, not borrow from it."

## Overview

The CapabilityGraph IR is the unification layer that merges three previously separate capability structures (v1.31 SemanticSummary, v1.32 CapabilityTopology, v1.34 ShardTopology) into one language-agnostic "Single Source of Truth." From this IR, all output targets are generated: Native (ELF), `.cap` audit manifest, and WIT (via CTL Mapper Fasa B).

## Design Philosophy

**Guard Rail:**
- WASM Guest = Unit Logik — NO direct hardware access
- All hardware access through Capability Gates → Host Reactor
- NO hidden scheduling — all async = explicit Reactor Events

## Architecture

```
v1.31 SemanticSummary ──┐
                         ├──► CapabilityGraph IR ──► Native (ELF)
v1.32 CapabilityTopology ─┤   (Single Source of    ──► .cap audit manifest
                         │    Truth)              ──► WIT (Fasa B)
v1.34 ShardTopology ────┘
```

## Core Types

### CompileTarget

```rust
pub enum CompileTarget {
    Native, // ELF — capability checks inlined
    Wasm,   // Sandboxed — maps to WASI via CTL
    All,    // Dual artifacts
}
```

### CapabilityRef

The canonical capability reference with optional WIT mapping (populated by Fasa B):

```rust
pub struct CapabilityRef {
    pub domain: String,           // Storage, Net, UI, HW, Audio, Crypto
    pub operation: String,        // Baca, Send, Papar, GPIO, etc.
    pub gate_type: GateType,      // DirectCall / Message / Hardware
    pub wit_mapping: Option<String>, // "wasi:filesystem/read" (Fasa B)
}
```

### IRServiceNode

Unified service node merging all three source structures:

```rust
pub struct IRServiceNode {
    pub id: u32,
    pub name: String,
    pub port: Option<u16>,
    pub requires: Vec<CapabilityRef>,
    pub provides: Vec<CapabilityRef>,
    pub handler: String,
    pub assigned_shard: Option<u32>,
    pub policy: String,
    pub effects: Capability,
    pub inline_cost: InlineCost,
    pub channels_used: Vec<String>,

```

### IRShardNode

```rust
pub struct IRShardNode {
    pub id: u32,

    pub budget_mb: u32,

```

### IRDoorEdge + IRGateEdge

```rust
pub struct IRDoorEdge {
    pub from_service: u32,
    pub to_service: u32,
    pub message_type: String,
    pub capacity: usize,

pub struct IRGateEdge {
    pub from: u32,

```

### CapabilityGraph — THE IR

```rust
pub struct CapabilityGraph {
    pub services: HashMap<u32, IRServiceNode>,
    pub shards: HashMap<u32, IRShardNode>,
    pub doors: Vec<IRDoorEdge>,
    pub gates: Vec<IRGateEdge>,
    pub target: CompileTarget,
    pub version: String,
}
```

## Verification: 6 Unified Checks

`CapabilityGraph::verify()` performs all checks from previous versions in one pass:

| Check | IRViolation | Source |
|---|---|---|
| Empty graph | `EmptyGraph` | New |
| WASM + hardware gate | `WasmHardwareGate { service, gate }` | v1.32 + v1.34 merge |
| Invalid shard assignment | `InvalidShardAssignment { service, shard_id }` | v1.34 |
| Unknown service in door | `UnknownServiceInDoor { service_id }` | v1.34 |
| Unknown service in gate | `UnknownServiceInGate { service_id }` | v1.32 |
| Empty shard | `EmptyShard { shard_id }` | v1.34 |
