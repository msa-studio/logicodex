# Logicodex Sharded Deterministic Reactor — v1.34.0-alpha

> **"No Global Lock, No Hidden State, No Magic."**
> Per-CPU-core reactor. Static affinity. Deterministic budgeting. Door-only cross-shard.

Architect: Mohamad Supardi Abdul (mymsastudio@gmail.com)

---

## Overview

v1.34.0-alpha adalah evolusi tertinggi setakat ini — transformasi dari **single-threaded Reactor** (v1.33) ke **multi-core Sharded Reactor** yang mengekalkan semua jaminan deterministik sambil mencapai near-linear scaling dengan CPU cores.

### The Sharded Reactor Manifesto

1. **Shard Isolation**: Setiap CPU Core = satu ReactorInstance + LocalPool. Tiada perkongsian memori.
2. **Affinity-Pinned**: Kompiler melakukan static mapping. Service WebServer "duduk" di Core 0. Tiada thread hopping.
3. **Deterministic Budgeting**: Setiap shard ada memory quota. Melampaui had = ditolak di peringkat Reactor.
4. **Cross-Shard = Door Only**: SPSC Message Passing. Forbidden by default — compile-time enforced.

## Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│                    STATIC MANIFEST                              │
│         (Compile-time: Service → Shard → Core)                  │
├───────────────┬───────────────┬─────────────────────────────────┤
│  Shard 0      │  Shard 1      │  Shard 2                        │
│  Core 0       │  Core 1       │  Core 2                         │
│               │               │                                 │
│ ┌───────────┐ │ ┌───────────┐ │ ┌───────────┐                  │
│ │Reactor    │ │ │Reactor    │ │ │Reactor    │                  │
│ │Instance   │ │ │Instance   │ │ │Instance   │                  │
│ │+ LocalPool│ │ │+ LocalPool│ │ │+ LocalPool│                  │
│ │+ Budget   │ │ │+ Budget   │ │ │+ Budget   │                  │
│ └───────────┘ │ └───────────┘ │ └───────────┘                  │
│ [WebServer]   │ [Metrics]    │ [Admin]                        │
│ [ApiGateway]  │ [Health]     │ [Config]                       │
│ Pool: 256MB   │ Pool: 64MB   │ Pool: 128MB                    │
└───────┬───────┴───────┬───────┴────────┬──────────────────────┘
        │               │                │
        └──── Door ─────┴─── Door ──────┘
      (SPSC Message Passing Only)
```

**Key**: Tiada shared state antara shards. Cross-shard communication HANYA melalui Door (SPSC Ring Buffer). Kompiler menolak sebarang percubaan komunikasi lain.

## Service Topology

Extends CapabilityTopology (v1.32) dengan Service Graph + Shard Mapping.

### ShardAssignment
```rust
pub struct ShardAssignment {
    shard_id: u32,        // 0, 1, 2, ...
    core_id: u32,         // CPU core (static mapping)
    services: Vec<String>, // Servis dalam shard
    budget_mb: u32,       // Memory quota
    gates: Vec<GateRef>,  // Gates yang dibenarkan
}
```

### ServiceGraph
```rust
pub struct ServiceGraph {
    nodes: HashMap<String, ServiceNode>, // Servis
    edges: Vec<CommEdge>,                // Komunikasi
}
```

### CommEdge — Two Types
- `CommType::Door` — SPSC async, **valid untuk cross-shard**
- `CommType::Direct` — Direct call, **HANYA intra-shard**

### ShardTopology::verify() — 6 Compile-Time Checks

| Check | Violation | Description |
|---|---|---|
| 1 | `UnassignedService` | Servis mesti di-assign ke shard |
| 2 | `DuplicateAssignment` | Servis tak boleh di >1 shard |
| 3 | `ForbiddenDirectCrossShard` | Cross-shard = Door only |
| 4 | `BudgetOverflow` | Penggunaan mesti dalam budget |
| 5 | `EmptyShard` | Shard tanpa servis = warning |
| 6 | `CoreConflict` | Satu core = satu shard |

## Manifest Format

```ldx
shard WebShard {
    core: 0,
    services: [WebServer, ApiGateway],
    budget_mb: 256,
    gates: [Net.Recv, Net.Send, Storage.Baca],
}

shard MetricsShard {
    core: 1,
    services: [MetricsCollector, HealthCheck],
    budget_mb: 64,
    gates: [Net.Recv],
}

// Cross-shard communication HANYA melalui Door
door WebShard → MetricsShard {
    message_type: MetricPayload,
    capacity: 1024,
}
```

**Anti-Pattern (ditolak oleh kompiler):**
```ldx
// ❌ ERROR: Cross-shard direct call forbidden!
let data = metrics_collector.get_data()  // Direct call cross-shard = REJECTED

// ✅ OK: Hantar melalui Door
metrics_door.send(MetricPayload { ... })  // SPSC message passing
```

## ShardedReactor

```rust
pub struct ShardedReactor {
    shards: Vec<ShardInstance>,
    topology: ShardTopology,
}

pub struct ShardInstance {
    shard_id: u32,
    core_id: u32,
    reactor: Reactor,
    pool: ShardLocalPool,
    services: Vec<String>,
}
```

**Lifecycle:**
1. `ShardedReactor::new(topology)` → verify topology → bina instances
2. `run()` → set CPU affinity → jalankan semua shards
3. `stop()` → hentikan semua shards (graceful)
4. `Drop` → shutdown all + tutup semua koneksi

## ShardLocalPool — Memory Budget

```rust
pub struct ShardLocalPool {
    budget_total: usize,  // Ditetapkan oleh kompiler
    budget_used: usize,   // Tracked secara ketat
}
```

- `acquire(size) → Result<AcquireResult, BudgetExceeded>`
- `release(buf)` → return ke pool
- `utilization() → f64` (0.0 - 1.0)

**Garanti**: Tiada allocation boleh melebihi budget. Reject berlaku sebelum memory habis.
